# プロジェクト作成画面 追加仕様（日本政策金融公庫：融資制度選択）

## 1. 目的
プロジェクト作成時に「調達元 = 日本政策金融公庫」を選んだ場合、ユーザーが **どの融資制度に申請するか** を迷わず決められるようにする。  
日本政策金融公庫サイトの「融資制度を探す」と同様に、**事業内容** および **利用目的・テーマ** を選択して候補を絞り込み、最終的に **融資制度（商品）** を選択できる仕様とする。

参照（公式の検索UI）  
- 「事業資金 融資制度を探す」：事業内容（生活衛生/サービス卸小売/その他）と、利用目的・テーマ（創業、新事業、セーフティネット、資本性ローン等）で検索できる。

---

## 2. 画面構成（Project Create Step）

### 2.1 既存：プロジェクト基本情報
- 調達元（Funding Source）
- 目標金額、調達時期、資金使途（目的）など

### 2.2 追加：日本政策金融公庫（JFC）専用ステップ
**表示条件**：`funding_source = JFC` の場合のみ表示

#### Step A：事業者区分（最上位）
最初に区分を選択する：
- `小規模事業者／個人事業主の方・中小企業の方`
- `農林漁業者・食品加工流通業の方`

> MVPでは、まず「小規模事業者／個人事業主の方・中小企業の方」を優先実装し、農林水産系は Phase2 としてもよい。

#### Step B：事業内容（Business Type）
チェックボックス（複数選択可）＋「すべてを含む／いずれかを含む」トグル（AND / OR）
- 生活衛生関係営業（※）
- サービス、卸・小売業
- その他の業種

**補足表示**：生活衛生関係営業の定義（ツールチップ/モーダル）

#### Step C：利用目的・テーマ（Purpose / Theme）
チェックボックス（複数選択可）＋「すべてを含む／いずれかを含む」トグル（AND / OR）
- 創業・創業後間もない方
- 新事業・多角化
- 事業拡大・生産性向上
- 事業承継・M&A
- ソーシャルビジネス
- 海外展開・事業再編
- SDGs
- 環境対策
- セーフティネット
- 事業再建・企業再生
- 財務体質の強化（資本性ローン）
- 災害関係（BCP含む）
- 商工会・商工会議所・生活衛生同業組合等の経営指導を受けている方
- その他のお使いみち
- 併用できる特例制度等

#### Step D：絞り込み結果（制度候補一覧）
- カード/リストで制度候補を表示（例：`新規開業・スタートアップ支援資金` など）
- 各候補には以下を表示
  - 制度名
  - 対象（国民生活事業 / 中小企業事業 等）
  - ざっくりの用途（設備/運転）
  - 主要な注意点（例：創業計画書が必要 等）
- クリックで「制度詳細（要約）」を右ペインまたはモーダルで表示

#### Step E：制度の確定（最終選択）
- 「この制度で進める」を押すと `selected_jfc_loan_program_id` をプロジェクトに保存
- 保存と同時に、**必要書類セット**（必須/推奨/参考）を生成・紐づけ
  - 生成ロジックは `JFC_LOAN_DOCUMENT_MATRIX.md` を参照

---

## 3. 絞り込みロジック（仕様）

### 3.1 入力
- `business_types[]`（複数） + `business_types_mode`（AND/OR）
- `themes[]`（複数） + `themes_mode`（AND/OR）
- `keyword`（任意：制度名・説明に対する全文検索）

### 3.2 候補データ（マスタ）
`JFC_LoanProgram` マスタを用意し、各制度にタグを持たせる：
- `id`, `name`, `segment`（国民生活/中小企業/農林水産）
- `business_type_tags[]`（生活衛生/サービス卸小売/その他）
- `theme_tags[]`（創業/新事業/生産性/承継/ソーシャル/海外/SDGs/環境/セーフティネット/再生/資本性/災害/指導/その他/特例）
- `reference_url`（公式ページ）
- `summary`（UI用短文）
- `doc_profile_id`（必要書類プロファイルへの参照）

### 3.3 絞り込み手順
1) `segment` を StepA で決定（または初期値）  
2) `business_types` を tag match  
3) `themes` を tag match  
4) `keyword` があれば `name/summary` に部分一致  
5) 上位表示ルール（任意）
   - テーマ一致数が多いほど上位
   - 「創業」選択時は創業向け制度を優先

---

## 4. UX要件（迷いを減らす）
- 画面上部に常に「いまの選択」が要約される（chips表示）
- 結果が0件の場合：
  - 選択の一部解除を促す（AND→ORの提案など）
  - キーワード検索の案内
- 結果が多すぎる場合：
  - 「よく選ばれる制度」を先頭に固定
  - 「創業」「セーフティネット」など代表テーマでサブフィルタを提案
- 制度確定後：
  - Document Checkerの左メニューに必要書類（必須/推奨/参考）が自動生成され、進捗が見える

---

## 5. 実装メモ（MVP→拡張）

### MVP（最短）
- マスタ制度数を絞る（例：創業/セーフティネット/資本性/ソーシャル/生活衛生/承継/海外 の代表制度）
- 絞り込みは `themes` のみ先行でもOK（事業内容は後で追加）
- 必要書類は「制度別に追加書類があるものだけ差分管理」し、ベースは共通セットを利用

### 拡張
- 公式サイトの検索結果（制度一覧）との同期（クローリング/手動更新）
- 「利用目的・テーマ」の選択に応じて追加質問を出す
  - 例：創業前か／創業後何期か、設備資金の有無、許認可の要否、生活衛生の該当業種など
- 中小企業事業・農林水産事業への対応
